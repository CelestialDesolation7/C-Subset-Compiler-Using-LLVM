# 线性扫描寄存器分配算法实现

## 自然语言概括

线性扫描寄存器分配请求函数：
如果程序请求一个存放数据的位置，调用该函数以决定存放到何处。

```bash
LinearScanRegisterAllocation (i) -> 一个空闲寄存器地址 | 一个空闲栈空间地址
	维护两个集合，初始为空：
		active    — 当前占用物理寄存器且在当前位置活跃的区间
		inactive  — 当前占用物理寄存器但在当前位置处于区间空洞的区间
	对于按起始点位置升序排列的，所有活动区间集合 AllInterval 中的每个活动区间 i 执行操作：
		调用函数 ExpireOldIntervals (i) 以释放所有不会再被使用的寄存器
		调用函数 HandleActiveInactiveTransitions (i) 处理区间空洞
		如果 (active 集合的元素数 = 寄存器总数 R)
		那么{ 
			// 发生溢出，需要将一个寄存器中的数据暂存到栈
			return SpillAtInterval (i)
		}
		否则{
			// 从空闲寄存器池中取出一个寄存器分配给活跃区间 i
			将 i 加入 active 集合中，按结束点位置升序排列；
			return 分配到的寄存器地址
		}
```

活动区间是一个Struct，名称为 i ，记录了以下数据：

- 一个对应的虚拟寄存器 %i ；
- 一系列活跃范围 `ranges`（每段为 `[start, end]`），一个区间可能包含**多段**活跃范围（多段之间的间隙称为**空洞**）；
- 它首次被使用的位置 StartPoint（= 第一段的 start）；
- 它最后一次被使用的位置 EndPoint（= 最后一段的 end）。

一个活动区间 i 在 active 集合中时，它必定满足以下性质：

- 编译器正在处理的指令位置（可以以行数表示），位于 i 的某一段活跃范围内；
- 其对应的 %i 的物理值位于物理寄存器中，而不是在栈上。

一个活动区间 i 在 inactive 集合中时，它满足：

- 编译器正在处理的指令位置，位于 i 的两段活跃范围之间的**空洞**中；
- 其物理寄存器仍被保留（不归还到空闲池），等待空洞结束后重新回到 active。

为了便于算法处理，我们要求：

- 名为 active 的活动区间集合，必须自发按结束点位置升序排列。

```bash
ExpireOldIntervals (i) -> void
	对于按结束点位置升序排列的 active 集合中的每一个活跃区间 j：
		如果 (j.EndPoint 在 i.StartPoint 的后方)
		那么{
			// j 还没有过期，不能移除
			// 由于 active 按结束点位置升序排列，后续的 j 也不会过期
			return
		}
		否则{
			// j 过期了，释放它的物理寄存器
			将 j 从 active 集合中移除
			将 j 正在占用的物理寄存器归还到空闲寄存器池中
			继续检查下一个 j 是否过期
		}
	return
```

```bash
HandleActiveInactiveTransitions (i) -> void
	// 步骤 1：将 active 中在当前位置进入空洞的区间移至 inactive
	对于 active 中的每一个活跃区间 j：
		如果 (j 在 i.StartPoint 处存在空洞 — 即该位置不被 j.ranges 中的任何一段覆盖)
		那么{
			将 j 从 active 移至 inactive（不归还物理寄存器）
		}

	// 步骤 2：处理 inactive 中的区间
	对于 inactive 中的每一个区间 k：
		如果 (k.EndPoint 在 i.StartPoint 之前)
		那么{
			// k 已彻底结束 → 释放寄存器
			将 k 从 inactive 中移除
			将 k 的物理寄存器归还到空闲池
		}
		否则 如果 (k 在 i.StartPoint 处恢复活跃 — 即该位置被 k.ranges 覆盖)
		那么{
			// k 的空洞结束 → 移回 active
			将 k 从 inactive 移回 active（按结束点升序插入）
		}
	return
```

 ```bash
 SpillAtInterval (i) -> 一个空闲寄存器地址 | 一个空闲栈空间地址
 	在 active 和 inactive 两个集合中，找到结束点最靠后的区间 spill
 	如果 (spill.EndPoint 在 i.EndPoint 的后方)
 	那么{
 		// spill 将晚于 i 被使用，更"不急切地"需要寄存器
 		// 将 spill 先放置到栈上代价较小
 		将 spill 溢出到内存，为其分配一个新的栈空间
 		从 active（或 inactive）中移除 spill
 		将 i 加入 active，按结束点升序排列
 		return spill 释放出的物理寄存器地址
 	}
 	否则{
 		// spill 将先于 i 被使用，更"急切地"需要寄存器
 		// 拒绝给 i 一个物理寄存器，只给其提供一个栈空间
 		return 一个空闲栈空间
 	}
 ```

以上函数的意义是实现"最小代价的溢出"。
这个算法以 **线性地遍历活跃区间** 为核心，保持 `active` 集合中为当前最活跃、最短命（即最早结束最后一次使用而可以释放空闲寄存器）的一组变量分配寄存器，同时通过 `inactive` 集合精确处理区间空洞，从而实现高效、实时的寄存器分配。
